# SignalForge FX v1.40 research model

v1.40 is a separate research fork of the v1.30 EA. It does not replace the
running v1.30 setup.

## FTMO deployment safety update

The current v1.40 deployment initializes every signal field explicitly and
blocks orders until all H1, H4, D1 and W1 indicator buffers are ready. Warm-up
exports safe zero values with the exact missing history component instead of
uninitialized telemetry.

It also adds an expected FTMO login/server lock, separate terminal and EA Algo
Trading permission checks, a startup notification diagnostic, WhatsApp relay
health telemetry, and a dashboard `DATA WARMING / ORDERS BLOCKED` state. These
changes do not alter the strategy's entry, exit, risk, regime or indicator
rules.

## Why this exists

A broader XAUUSD test showed that the earlier model can perform well in one
gold regime and fail in another. v1.40 adds regime filters so signals are only
allowed when the local H1 setup agrees with broader market structure.

## Near-signal phone alert

When either direction reaches at least 4/5 base checks on a newly closed H1
candle, v1.40 can send a `SETUP FORMING` push notification. The message includes
the directional bias and regime-filter count and explicitly says `NO TRADE YET`.
`InpEnableNearSignalPush` and `InpEnablePush` must both be enabled. Automatic
orders still require all 5/5 base checks and all 6/6 filters.

## Trade Readiness Center

The EA writes `SignalForge\strategy.json` in the MT5 Common Files directory.
SignalForge Monitor merges that snapshot into dashboard schema 4. The frontend
therefore shows the EA's exact BUY/SELL base counts, D1/W1 regime filters,
next H1 close, auto-trading state, risk setting, and last phone-push result.
The simplified monitor signal is used only as a fallback when the v1.40
strategy snapshot is unavailable.

## News Risk Guard

v1.40 can read high-impact USD events from MT5's native economic calendar.
`InpEnableNewsAdvisory` enables a 30-minute pre-event warning and a 15-minute
post-event caution window by default. One phone advisory is queued per event,
and schema 4 exposes the event, countdown, status, and calendar health to the
dashboard.

This feature is advisory only. It does not block, open, close, or modify trades.
Historical news data must be frozen separately before the rule can be tested
reproducibly in Strategy Tester.

## Added filters

- D1 EMA 50 / EMA 200 regime alignment
- D1 regime EMA slope confirmation
- W1 EMA 20 / EMA 50 macro-regime alignment
- W1 macro EMA separation filter
- ATR-percent volatility bounds
- EMA 20 / EMA 50 separation measured against ATR
- H1 slow-EMA slope confirmation
- ATR breakout buffer beyond the prior channel
- Optional rollover-hour avoidance

## Validation rule

Do not attach v1.40 for demo trading unless it passes:

1. Full-period XAUUSD real-tick test.
2. Held-out forward-period test.
3. Stress test with higher spread/slippage.
4. At least 60-90 days of unchanged demo observation.
